# 为什么选择静态博客

最近我重新设计了我的博客系统，从原来的数据库驱动（Prisma + PostgreSQL）转变为纯静态文件方案。这篇文章记录了我的思考过程。

## 动态博客的问题

### 复杂性

数据库驱动的博客需要：

- 数据库连接和管理
- 用户认证系统
- API 路由和权限控制
- 部署时的数据库迁移

### 成本

- 数据库托管费用
- 更高的服务器资源需求
- 备份和维护成本

### 安全风险

- SQL 注入风险
- 认证系统漏洞
- 需要持续安全更新

## 静态博客的优势

### 简单

```
内容 → 构建 → 部署 → CDN
```

没有运行时依赖，部署就是一堆静态文件。

### 快速

- 预渲染的 HTML
- 可以直接部署到 CDN
- 首屏加载极快

### 安全

- 没有数据库 = 没有 SQL 注入
- 没有认证 = 没有账户被盗
- 攻击面极小

### 免费托管

可以部署到：

- Vercel
- Cloudflare Pages
- GitHub Pages
- Netlify

全部免费！

## 我的新方案

使用 **Contentlayer** + **Next.js**：

1. 文章用 MDX 格式写在 `content/posts/` 目录
2. Contentlayer 将 MDX 编译成类型安全的 JSON
3. Next.js 在构建时生成静态页面
4. 部署到 Vercel

需要新文章？直接创建 `.mdx` 文件，提交代码，自动部署。

## 缺点

当然，静态博客也有局限：

| 局限 | 说明 |
|------|------|
| 无动态数据 | 评论、点赞需要第三方服务 |
| 构建时间 | 文章多了构建会变慢 |
| 实时性 | 内容更新需要重新部署 |

但对于个人博客，这些都不是问题。

## 结论

**简单即美。**

对于个人博客这种内容为主的网站，静态方案是更务实的选择。把复杂留给需要它的地方。
